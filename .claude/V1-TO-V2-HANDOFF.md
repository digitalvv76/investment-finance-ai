## 重申：改动归属判断（每次动手前必读）

| 触发 | 正确做法 |
|------|----------|
| 用户反馈推送质量问题 | **V1 窗口观察 → V1 诊断 → V1 出方案 → 写交接给 V2** |
| 纯技术问题（架构/代码/测试/部署） | V2 自行决策 |

| 部署生产 | 仅 V2 执行（§1） |
| 不确定归属 | **先提醒用户**，不直接执行（§6） |
