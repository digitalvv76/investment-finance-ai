三、LLM 评估 Prompt 设计

System Prompt

You are a senior macro analyst at a top-tier hedge fund. Your job is to evaluate the potential MARKET IMPACT of a financial news event. You think in terms of: 1. EVENT TYPE & INHERENT SIGNIFICANCE - Monetary policy (FOMC, rate decisions, forward guidance) → highest impact - Geopolitical shocks (war, sanctions, trade war escalation) → very high impact - Major macro data surprises (CPI, NFP, retail sales) → high impact - Corporate earnings / product launches (mega-cap only) → moderate impact - Routine data / minor policy → low impact 2. SURPRISE MAGNITUDE - How far does the actual figure deviate from expectations? - A 0.1% CPI miss is noise; a 0.5% miss is a regime change signal. 3. MARKET BREADTH - Single stock → sector → broad index → multi-asset (equities+bonds+FX+commodities) - Wider breadth = higher impact 4. HISTORICAL PRECEDENT - Has a similar event occurred in the past 2 years? What was the market reaction? - Use your training knowledge to contextualize. 5. CURRENT MARKET CONTEXT - Is the market positioned for this? (crowded trades amplify moves) - Current VIX / Fear & Greed regime (fear amplifies negative news) OUTPUT FORMAT (strict JSON): { "impact_score": 0-100, "confidence": 0-100, "event_category": "monetary|geopolitical|macro_data|corporate|regulatory|other", "surprise_level": "expected|minor_surprise|major_surprise|shock", "breadth": "single_stock|sector|broad_market|cross_asset", "reasoning_chain": [ "Step 1: Event type analysis...", "Step 2: Surprise magnitude...", "Step 3: Market breadth...", "Step 4: Historical context...", "Step 5: Current regime..." ], "similar_historical_events": ["Event A (date): impact description", ...], "expected_sectors_affected": ["Technology", "Financials", ...], "expected_asset_moves": { "equities": "direction magnitude", "bonds": "direction magnitude", "fx": "direction magnitude", "commodities": "direction magnitude" }, "calibration_note": "Based on past assessments, this evaluator tends to [over/under]estimate [type] events by [X] points" }

五、自学习机制

📥 学习循环

1. LLM 评估新闻 → 预测 impact_score
▼ 等待市场反应
2. 采集实际市场数据 (SPX Δ, VIX Δ, sector Δ)
▼ 计算误差
3. Δ = |predicted − actual| → 存入 calibration_db
▼ 定期分析
4. 提取校准提示 → 注入下轮 LLM Prompt

📊 学习维度

维度跟踪指标校准动作
事件类型偏差 monetary ±X, macro ±Y 调整该类型 baseline
惊喜幅度校准 over-estimate minor surprises 提高"shock"门槛
广度误判 预测 cross_asset 实际 single_stock breadth 降权
情绪状态 fear 时高估, greed 时低估 VIX 调整系数
时效衰减 评估越旧越不准 加权近期样本

六、数据存储

新增 DB 表

CREATE TABLE impact_assessments ( id INTEGER PRIMARY KEY, news_id INTEGER, -- FK → news_items impact_score REAL, -- 0-100 LLM 预测 confidence REAL, -- 0-100 event_category TEXT, surprise_level TEXT, breadth TEXT, reasoning_chain TEXT, -- JSON array similar_events TEXT, -- JSON array expected_moves TEXT, -- JSON calibration_note TEXT, created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ); CREATE TABLE impact_outcomes ( id INTEGER PRIMARY KEY, assessment_id INTEGER, -- FK → impact_assessments spx_change_pct REAL, -- 实际标普变动 vix_change_pct REAL, -- 实际 VIX 变动 sector_changes TEXT, -- JSON {sector: change_pct} actual_score REAL, -- 事后归一化评分 0-100 collected_at TIMESTAMP ); CREATE TABLE calibration_state ( id INTEGER PRIMARY KEY, category TEXT UNIQUE, -- event_category or 'global' bias REAL, -- 平均过估/低估 (正=过估) sample_count INTEGER, last_updated TIMESTAMP );

七、实际影响归一化

如何将市场数据转为 "actual_score" (0-100)

维度公式权重
标普变动 min(|SPX_Δ%| / 0.03, 1.0) × 100 40%
VIX 变动 min(|VIX_Δ%| / 0.15, 1.0) × 100 25%
板块广度 (affected_sectors / 11) × 100 20%
跨资产联动 (bonds_moved + fx_moved + commodities_moved) / 3 × 100 15%

例: SPX -2.1%, VIX +18%, 5/11 板块跌 >2%, 债券异动 = (0.7×40)+(0.72×25)+(0.45×20)+(0.33×15) = 60 → moderate-high

十、与现有系统的隔离

⚠️ 新方案零影响现有手机告警
  • ImpactEvaluator 不连接 AlertDispatcher
  • 新评分 不写入 news_items.priority_score 字段
  • 新表 独立存储，不影响现有查询
  • Dashboard 通过 /impact 子页面访问，不混入主页面
  • 如需未来合并，只需加一个开关：if impact_score > 80 → AlertDispatcher.classify()
