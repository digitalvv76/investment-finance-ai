# 资金流背离信号模型 — 正式版

> 版本: 2026-07-16 · 数据源: Futu OpenD `get_capital_flow` · 覆盖: 美股 + 港股

---

## 一、核心信念

**不看单日涨跌，看涨跌与特大单流向是否一致。**
一致 = 趋势延续；背离 = 拐点临近。

一切技术指标都可以造假，但大资金的真金白银不会说谎。特大单（单笔 ≥ 100 万元）无法被机构隐藏——它是整个分析体系的锚点。

---

## 二、订单分类体系

| 层级 | 字段名 | 来源 | 定义 | 市场角色 |
|:---|:---|:---|:---|:---|
| **★ 特大单** | `super_big_net` | Futu `super_in_flow` | 单笔 ≥ 100 万元 | **机构/主力** — 趋势发动者，锚点 |
| 大单 | `big_net` | Futu `big_in_flow` | 单笔 20-100 万元 | 游资/大型私募 — 趋势加速器 |
| **主力（合计）** | `main_net` | **计算**: `super + big` | 特大单+大单 | 防机构拆单规避检测 |
| 中单 | `mid_net` | Futu `mid_in_flow` | 单笔 4-20 万元 | 中大户 — 情绪缓冲垫 |
| 小单 | `small_net` | Futu `sml_in_flow` | 单笔 < 4 万元 | 散户 — 反向指标 |

### 为什么主力 = 特大单 + 大单？

机构可以通过将一笔 500 万的订单拆成 25 笔 20 万的单子来规避"特大单"检测。
将特大单和大单合并统计，能堵住这个漏洞。这是 V2.1 P0 的核心修正。

### 那 Futu 的 `main_in_flow` 呢？

富途 API 返回一个 `main_in_flow` 字段（算法识别的"Block Orders"），但这是富途自己
的算法定义，标准不透明。本模型不使用该字段。

## 三、信号框架

```
                         ┌─────────────────────┐
                         │   ★ 特大单 (anchor)  │
                         │   方向判断           │
                         └─────────┬───────────┘
                                   │
              ┌────────────────────┼────────────────────┐
              │                    │                    │
        股价跌 + 特大单流入    股价涨 + 特大单流出    量价同向
              │                    │                    │
         🟢 底背离             🔴 顶背离            ❌ 不推
         "主力吸筹"           "主力出货"
              │                    │
              └──────────┬─────────┘
                         │
              辅助确认（四法则）
              ┌──────────┼──────────┐
              │          │          │
           A.小单反向  B.连续性   C.主力占比
          (散户情绪)  (持续方向) (参与强度)
```

## 四、背离信号定义

### 底背离 — 🟢 重点关注

> 股价暴跌 + 特大单逆势净流入（或仅微幅流出）→ 主力在吸筹

| 股价表现 | 特大单表现 | 信号强度 | 结论 |
|:---|:---|:---:|:---|
| 跌 > 10%（单日或 3 日累计） | 净流入 > 0 | ★★★ | 强底背离，短期反弹概率极大 |
| 跌 > 5% | 净流入 > 0 | ★★ | 底背离，关注 |
| 跌 > 10% | 净流出 < 总流出 10%（几乎没跑） | ★ | 弱底背离，主力没跟着砸 |

### 顶背离 — 🔴 风险预警

> 股价大涨 + 特大单逆势净流出（或仅微幅流入）→ 主力在出货

| 股价表现 | 特大单表现 | 信号强度 |:---|
| 涨 > 10%（单日或 3 日累计） | 净流出 > 0 | ★★★ | 强顶背离，短期回撤概率极大 |
| 涨 > 5% | 净流出 > 0 | ★★ | 顶背离，警惕 |
| 涨 > 10% | 净流入 < 总流入 10%（几乎没跟） | ★ | 弱顶背离，主力没参与狂欢 |

### 不产生信号

| 场景 | 原因 |
|:---|:---|
| 股价涨 + 主力流入 | 量价齐升，正常追涨 |
| 股价跌 + 主力流出 | 量价齐跌，正常抛售 |
| 股价波动 < 5% | 幅度不够 |
| 主力占比 < 8% | 机构参与度太低，噪音 |

## 五、辅助确认法则

背离信号出现后，依次检查以下四项来确认或降级。

### 法则 A：小单反向确认

| 组合 | 效果 |
|:---|:---|
| 底背离 + 小单净流出（散户恐慌抛售） | ⬆️ 信号增强 — 底部确认 |
| 顶背离 + 小单净流入（散户疯狂追涨） | ⬆️ 信号增强 — 顶部确认 |
| 背离 + 小单同向（散户跟机构走） | ⬇️ 信号降级 — 可能是真趋势 |

### 法则 B：连续性确认

| 特大单 3 日方向 | 股价 | 含义 |
|:---|:---|:---|
| 连续净流入 | 横盘或微跌 | 主力在持续建仓 |
| 连续净流出 | 横盘或微涨 | 主力在持续减仓 |
| 进出交替 | — | 背离不成立，主力在做波段 |

### 法则 C：主力占比定强度

主力占比 = (特大单+大单) / |总成交额| × 100%

| 占比 | 调整 |
|:---|:---|
| < 1% | 机构参与度低，信号 **降一级** |
| 1-5% | 正常参与，维持 |
| 5-10% | 机构主导，信号 **升一级** |
| > 10% | 极端信号，警惕数据异常 |

## 六、分析流程

```
第〇步：排除噪音日
  ├── 财报日 ±1天 · FOMC/CPI/非农 · 期权到期 · VIX>35
  └── → 降权但不作废

第一步：趋势定位
  └── MA7/MA25/MA99 · 52周高低点 · 均线乖离率

第二步：搜寻背离
  └── 逐日扫描 涨幅 vs 特大单方向

第三步：辅助确认
  └── 小单反向 · 连续性 · 主力占比 · 板块确认 · 关键价位

第四步：情景推演
  └── 三种情景(乐观/中性/悲观) + 概率 + 具体验证条件
```

## 七、推送策略

| 信号强度 | 通道 | 文案格式 |
|:---|:---|:---|
| ★★★ 强背离 | 📱 Pushover + TG 铃声 | "🔵 NVDA 底背离：3日跌14.8%，特大单逆势净流入，主力占比18.5%" |
| ★★ 标准背离 | 💬 TG 响铃 | 同上 |
| ★ 弱背离 | 💬 TG 静默 | "📊 AAPL 弱底背离：3日跌10.2%，特大单微幅流出，待观察" |
| 无背离 | ❌ | 仅入库 |

### 推送语义（V2.1 P0）

- ~~买入信号~~ → **重点关注**
- ~~卖出信号~~ → **风险预警**

### 财报静默期（V2.1 P0）

财报后 5 个日历日内的信号仅入库，不推送任何通道。

## 八、代码层信号计算

### `compute_divergence_signal()` 算法

```
输入: List[FundFlowDay] (≥3日)
输出: {signal, strength, detail, details}

1. 取最近 3 日
2. 逐日比较:
   - price_delta = day[i].change_pct - day[i-1].change_pct
   - flow_delta = day[i].super_big_net - day[i-1].super_big_net  ← 特大单锚点
3. 背离判定:
   - price_delta>0 && flow_delta<0 → 顶背离计数+1
   - price_delta<0 && flow_delta>0 → 底背离计数+1
4. 信号输出:
   - ≥2日背离 → bullish/bearish_divergence
   - 1日背离 → warning
   - 同向 → confirmation
   - 其他 → none

details:
  - cum_main_3d: 主力(super+big) 3日累计
  - cum_super_big_3d: 特大单 3日累计
  - latest_main_pct: 最新日主力占比
  - continuity: continuous_inflow | continuous_outflow | mixed
  - participation: extreme(>15%) | strong(>8%) | normal(>3%) | low
```

## 九、数据映射

### Futu API → 本模型

| Futu `get_capital_flow` 字段 | 本模型字段 | 用途 |
|:---|:---|:---|
| `super_in_flow` | `super_big_net` | ★ 锚点 — 背离方向 |
| `big_in_flow` | `big_net` | 参与主力计算 |
| — | `main_net` | **计算**: `super + big` |
| — | `main_pct` | **计算**: `(super+big)/abs(total)*100` |
| `mid_in_flow` | `mid_net` | 辅助 — 散户情绪 |
| `sml_in_flow` | `smal_net` | 辅助 — 散户情绪 |
| `in_flow` | (不存储) | 仅用于计算 main_pct 的分母 |
| `main_in_flow` | ❌ 不使用 | Futu 自有算法标准，不透明 |

## 十、特殊标的处理

### 杠杆 ETF（SOXL、TQQQ、SQQQ 等）
- 背离阈值提高到 ±15%（而非 ±10%）
- 优先用底层指数（SOXX、QQQ）资金流交叉验证
- 小单占比 > 80% 为额外散户情绪信号

### 小盘股 / 低流动性
- 绝对金额门槛降低（日成交 < $10M 时机构进出受限）
- 重点看主力占比而非绝对金额
- 信号置信度自动降一级

### 港股
- 港股通资金流作为北水补充确认
- 波动更大，背离阈值放宽到 ±15%
